# Scope — права доступа

Scope определяет к каким данным и действиям получает доступ приложение.  
Пользователь видит список запрошенных прав на странице подтверждения и может отклонить запрос.

> Запрашивай только те scope которые реально нужны приложению.

## Доступные scope

| Scope            | Название              | Что включает |
|------------------|-----------------------|--------------|
| `users`          | Профиль               | Чтение и редактирование профиля, подписки, блокировки, поиск пользователей |
| `posts`          | Публикации            | Чтение, создание, редактирование постов, лайки, репосты, комментарии |
| `comments`       | Комментарии           | Редактирование и удаление своих комментариев, лайки |
| `notifications`  | Уведомления           | Просмотр уведомлений и управление настройками |
| `files`          | Файлы                 | Загрузка и удаление файлов от имени пользователя |
| `reports`        | Жалобы                | Отправка жалоб на контент |
| `hashtags`       | Хэштеги               | Просмотр публикаций по хэштегам и трендов |
| `search`         | Поиск                 | Глобальный поиск по платформе |
| `subscription`   | Подписка              | Просмотр статуса подписки и управление методами оплаты |
| `verification`   | Верификация           | Просмотр статуса верификации аккаунта |
| `platform`       | Платформа             | Просмотр журнала обновлений |

## Примеры наборов

**Сайт статистики** — только чтение профиля:
```
scope: "users"
```

**Лента и взаимодействие:**
```
scope: "users posts comments notifications"
```

## Запрещённые действия

Независимо от scope, следующие действия всегда заблокированы:

- `DELETE /users/me` — удаление аккаунта пользователя

## Проверка scope в токене

```ts
const payload = itd.decodeToken(token);

if (payload.scope.includes("posts")) {
  // можно работать с постами
}
```
